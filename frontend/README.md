# Solace — Frontend

React frontend for the Solace emotional intelligence chatbot. Communicates with the FastAPI backend via REST and Server-Sent Events. In production it is compiled and served statically by the backend. In development it runs on its own port with hot-reload.

---

## Requirements

- Node.js 18 or later
- npm 9 or later

---

## Setup

```bash
cd frontend
npm install
```

---

## Development

Start the Vite dev server with hot-reload:

```bash
npm run dev
```

The app runs at `http://localhost:5173`. API requests (`/chat`, `/chat/stream`) are proxied to the backend at `http://localhost:8000`, which must also be running.

---

## Production Build

```bash
npm run build
```

Output goes to `frontend/dist/`. The FastAPI backend serves this directory automatically — no separate web server is needed.

To preview the production build locally before deploying:

```bash
npm run preview
```

## Features

- **Streaming responses** — messages render word by word as the LLM outputs them, extracted in real time from partial JSON via regex
- **Emotion backgrounds** — animated color blobs shift based on the detected emotion in each response
- **Mood sparkline** — a mini chart shows emotional valence trend across the conversation
- **Breathing exercise** — fullscreen guided breathing overlay (4-7-8 pattern), opened with the `/` key or toolbar button
- **Ambient sound** — rain, warm hum, or distant waves generated by the Web Audio API; no audio files required
- **Session history** — past conversations saved to `localStorage` and accessible from the sidebar; titles are editable via double-click
- **Theme** — light and dark modes, persisted to `localStorage`, respects `prefers-color-scheme` on first load
- **Font size** — three sizes, persisted to `localStorage`
- **Export** — save conversation as `.txt`, copy to clipboard, or print as PDF
- **PWA** — installable as a Progressive Web App with offline support via a service worker
- **Accessibility** — ARIA live regions, focus management, reduced-motion support

## Environment Variables

The frontend has one optional environment variable:

| Variable | Default | Description |
|---|---|---|
| `VITE_API_URL` | `""` (relative) | Base URL for API requests. Leave empty when the frontend is served by the backend on the same origin. Set to `http://localhost:8000` for local dev if the Vite proxy is not configured. |

Create a `.env.local` file in the `frontend/` directory to override:

```
VITE_API_URL=http://localhost:8000
```

In production (single-server deployment), this variable should be left unset so all requests go to the same origin.
